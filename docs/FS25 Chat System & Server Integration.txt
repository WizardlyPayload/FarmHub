# Developer Handover: FS25 Chat System & Server Integration

**Target Environment:** GIANTS Engine 10 (Farming Simulator 25)
**Language:** Lua, XML, WebAPI
**Topic:** Chat Architecture, Multiplayer Networking, and Dedicated Server (G-Portal) Compatibility

---

## 1. Architecture Overview: The Chat Lifecycle

In the GIANTS Engine, the chat system operates on a strict **Client -> Server -> Clients** broadcast loop. To manipulate the chat effectively, code must be injected at the correct stage:

1.  **Input (Client):** User types in the UI and presses Enter.
2.  **Transmission (Client to Server):** The client packages the string into a `ChatEvent` and sends it to the server.
3.  **Processing (Server):** The server receives the event, verifies the sender, and logs it.
4.  **Broadcast (Server to Clients):** The server duplicates the event and sends it to all connected clients.
5.  **Render (Clients):** Each client receives the broadcast and draws the text on their screen.

## 2. Multiplayer Topology Matrix

The game executes logic differently based on the server type. A mod designed for chat must handle the "Headless" state of dedicated servers to prevent fatal null-pointer exceptions.

| State | g_server | g_client | g_dedicatedServer | UI Object (chatWindow) |
| :--- | :--- | :--- | :--- | :--- |
| **Singleplayer** | True | True | False | Exists |
| **Listen Server (Host)** | True | True | False | Exists |
| **Dedicated (G-Portal)** | True | False | True | **NIL (Does not exist)** |

## 3. Core Implementation Modules

### Module A: Client-Side Command Interception
Captures user input and "swallows" it if it starts with a command prefix (e.g., `/pay`), preventing it from appearing in public chat.

```lua
-- Hooking the standard chat window
local original_onSendChat = ChatWindow.onSendChat

ChatWindow.onSendChat = function(self, text)
    if string.sub(text, 1, 1) == "/" then
        local commandText = string.sub(text, 2)
        
        if g_client ~= nil then
            -- Send secure event to the server
            g_client:getServerConnection():sendEvent(MyCustomChatEvent.new(commandText))
        end
        
        -- Return early to 'swallow' the message
        return 
    end
    original_onSendChat(self, text)
end
```

### Module B: The Dedicated Server "Safe" Broadcast
On G-Portal servers, `g_currentMission.chatWindow` is `nil`. Calling functions on it will crash the server.

```lua
function MyMod:safeBroadcast(message)
    if g_server ~= nil then
        -- 1. Sync across network to all players
        g_server:broadcastEvent(ChatEvent.new(message, "System", true))
        
        -- 2. Local Rendering (Skip if Dedicated/G-Portal)
        if not g_dedicatedServer and g_currentMission.chatWindow ~= nil then
             g_currentMission.chatWindow:addMessage(message, "System", true)
        end
        
        -- 3. Print to G-Portal console log
        print("CHAT_LOG: " .. message)
    end
end
```

## 4. Server-Side Processing & Security
Admin commands must be validated on the server side to prevent exploits.

```lua
function MyMod:executeServerCommand(commandArgs, senderConnection)
    -- Security Check: Is the sender an Admin?
    if not senderConnection:getIsServerAdmin() then
        g_server:broadcastEvent(ChatEvent.new("Permission denied.", "System"), nil, senderConnection, nil)
        return
    end

    -- Process Example: /giveMoney
    if commandArgs[1] == "giveMoney" then
        local amount = tonumber(commandArgs[2])
        g_currentMission:addMoney(amount, senderConnection:getFarmId(), MoneyType.OTHER, true, true)
        print(string.format("ADMIN: %s spawned %s money.", senderConnection:getLogName(), amount))
    end
end
```

## 5. Network Event Class (Synchronization Template)
Required for transmitting custom data between clients and the server instance.

```lua
MyCustomChatEvent = {}
local MyCustomChatEvent_mt = Class(MyCustomChatEvent, Event)
InitEventClass(MyCustomChatEvent, "MyCustomChatEvent")

function MyCustomChatEvent.emptyNew() return Event.new(MyCustomChatEvent_mt) end

function MyCustomChatEvent.new(commandString)
    local self = MyCustomChatEvent.emptyNew()
    self.commandString = commandString
    return self
end

function MyCustomChatEvent:readStream(streamId, connection)
    self.commandString = streamReadString(streamId)
    self:run(connection)
end

function MyCustomChatEvent:writeStream(streamId, connection)
    streamWriteString(streamId, self.commandString)
end

function MyCustomChatEvent:run(connection)
    if not connection:getIsServer() then
        -- Execute command on the server
        MyMod:executeServerCommand({self.commandString}, connection)
    end
end
```

## 6. External Integration (WebAPI & Logs)

1.  **G-Portal Log Mirroring:** All `print()` statements appear in the G-Portal `server.log`. External Discord bots use this to mirror in-game chat.
2.  **WebAPI Injection:** Messages from the G-Portal dashboard enter directly as `ChatEvent`. If filtering is needed, hook `g_server:broadcastEvent`.
3.  **Headless Constraints:** The server machine has no keyboard input. All logic must be triggered by clients via `Event` classes.
