Where things run

Farm Dashboard (Electron, usually port 8766) often runs on your PC next to the game. AI Farm Manager (this page) may run on a cloud VPS. Your PC calls the VPS (robot panel → AI backend URL) — no inbound ports required on your PC if you use PC push mode: set DASHBOARD_PUSH_MODE=1 in this server’s environment, leave Farm Dashboard JSON URL empty, and in Farm Dashboard enable Push snapshots to AI server (same panel). Snapshots are POSTed outbound from your PC. Alternatively the VPS can use G-Portal FTP or (if you accept it) an HTTP URL/tunnel to your PC.

Bot & integrations

Farm Dashboard link key — set in backend/.env only as FARMDASH_INTEGRATION_KEY=…, and paste the same value into the Farm Dashboard app (robot button → Farm Dashboard link key). Not your LLM API keys below. Not on this form.

Used to pull /api/data for the bot and to list servers from the same host’s /api/servers. Must be reachable from this backend: plain 127.0.0.1 only works when AI Farm Manager runs on the same OS as Farm Dashboard (not inside a remote VPS or container — use host.docker.internal, your LAN IP, FTP mode, or a tunnel).

Use an id from Google’s API for your key (e.g. gemini-2.5-flash, gemini-2.0-flash). If you see HTTP 404, try GEMINI_REST_API_VERSION=v1beta or another model name.

Loads the latest snapshot, picks the first owned field for that farm, then runs the single-field consultant (same as ?fieldRef=<farmlandId>) so the reply is the suggested next job on that parcel. May take 10–60s. Server API keys only (not BYOK).

Multi-server bot (Farm Dashboard)

Add one profile per dedicated game server. Each profile gets its own server_token and message queue. Map the Farm Dashboard server id so the LLM pulls /api/data?serverId=… for the correct save. Do not edit XML by hand: use Download XML per profile, or Farm Dashboard’s Install config (writes modsSettings/ai_farm_manager_config.xml for you). Set PUBLIC_BASE_URL in .env so generated files point at your real API (HTTPS) hostname.
